# CA Dashboard Frontend - Comprehensive Test Cases

## Project Overview
CA Dashboard is a React-based asset management application built with Vite, featuring:
- Authentication system with JWT tokens
- Asset management (CRUD operations)
- Financial year management
- Multiple asset types, manufacturers, and vendors
- Dashboard with analytics and charts
- Responsive design with shadcn/ui components

## Test Environment Setup
- **Frontend URL**: http://localhost:5173 (Vite dev server)
- **Backend API**: http://192.168.0.190:8000/api
- **Test Browser**: Chrome 115+ (as specified in footer)
- **Test Data**: Use organization ID 962834 for testing

## Test Execution Guidelines

### Pre-Test Setup
1. Ensure backend server is running on http://192.168.0.190:8000
2. Clear browser cache and localStorage
3. Use Chrome 115+ browser for primary testing
4. Have test data ready (organization ID: 962834)

### Test Data Requirements
- Valid user credentials for authentication
- Sample asset data for CRUD operations
- Multiple financial years for testing
- Various asset types, manufacturers, and vendors

### Test Environment
- **Frontend**: Vite development server (http://localhost:5173)
- **Backend**: Django REST API (http://192.168.0.190:8000/api)
- **Database**: PostgreSQL (backend managed)
- **Browser**: Chrome 115+ (primary), Firefox, Safari, Edge

### Test Execution Order
1. Authentication tests first
2. Basic functionality tests
3. Integration tests
4. Performance tests
5. Security tests
6. Browser compatibility tests

### Defect Reporting
For each failed test case, report:
- Test Case ID
- Steps to reproduce
- Expected vs actual result
- Screenshots (if applicable)
- Browser and version
- Console errors (if any)

### Test Completion Criteria
- All critical test cases (AUTH, ROUTE, ASSET) must pass
- 95% of functional test cases must pass
- No critical security vulnerabilities
- Performance within acceptable limits
- Cross-browser compatibility verified

---

## Notes
- This test suite covers the frontend React application only
- Backend API testing should be done separately
- Some tests may require backend modifications for proper testing
- Regular regression testing recommended after each release
- Test data should be reset between test runs for consistency
